# ADR 0002 — Arquitetura ECS própria e enxuta

**Status:** Aceito · **Data:** 2026-06-15

## Contexto
Mundo aberto + hordas + invocações + projéteis geram muitas entidades
heterogêneas. Precisávamos de uma forma escalável e flexível de organizar
estado e lógica, sem acoplar dados a hierarquias de classe.

## Decisão
ECS próprio e minimalista (`src/core/ecs/World.js`):
- Entidade = id inteiro; componente = objeto de dados puro guardado em `Map`
  por tipo; sistema = função `(game, dt)`.
- Query por interseção de conjuntos, começando pelo menor store.
- Remoção diferida (`flushDestroyed`) para não mutar durante a iteração.
- Event bus simples no World para comunicação entre sistemas.

Evitamos uma lib de ECS de terceiros para manter controle, zero dependências e
uma curva de leitura baixa no protótipo.

## Consequências
- Fácil adicionar comportamento compondo componentes.
- Query O(n) ingênua; quando hordas crescerem, trocar broadphase de colisão por
  spatial hash (ver backlog M9). Para a escala atual é suficiente.
